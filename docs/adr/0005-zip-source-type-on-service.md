# Zip 来源作为 Service 的扩展而非独立实体

Pagekit 原先只支持 Git 仓库作为静态资源来源。为支持「上传 zip 解压后部署」，我们在现有 **Service** 上增加 `source_type`（`git` | `zip`），而不是引入 Upload/Site 等并列实体。

Zip 服务通过 **Stored Archive（存档包）** 持有最新 zip，经独立上传接口写入；发布与更新从存档包解压，复用与 Git 服务相同的发布目录语义和原子文件切换。来源类型在创建时选定且不可变更；zip 原样解压（不自动剥单层目录）；上传做 Zip Slip 校验并有可配置大小上限（默认 50MB）。更新时可选择是否上传新 zip——无新包则从现有存档包按当前配置重新解压。

曾考虑独立实体（边界清晰但重复 CRUD/发布逻辑）及创建时内嵌 multipart（耦合上传与元数据）。统一为 Service 后，列表、二维码、日志、发布位置等能力零重复；Git 专有字段（凭证、分支、webhook）仅对 git 类型生效。
